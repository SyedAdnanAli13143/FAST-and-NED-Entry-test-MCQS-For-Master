# Software Development Models (5 Questions)

**Q1.** Which SDLC model follows a strict, sequential, phase-by-phase approach (requirements → design → implementation → testing → deployment) with no going back?
A) Agile  B) Waterfall  C) Spiral  D) Scrum
**Answer: B) Waterfall**

**Q2.** Which model is best suited for projects with **rapidly changing requirements**?
A) Waterfall  B) Agile  C) V-Model  D) None of these
**Answer: B) Agile**

**Q3.** In Agile (specifically Scrum), work is divided into short, fixed-length cycles called:
A) Phases  B) Sprints  C) Milestones  D) Releases
**Answer: B) Sprints**

**Q4.** Which Agile-related framework uses a visual board with columns like "To Do / In Progress / Done" to track work?
A) Scrum  B) Waterfall  C) Kanban  D) Spiral
**Answer: C) Kanban**

**Q5.** In Scrum, the short daily meeting where team members share progress and blockers is called:
A) Sprint Review  B) Daily Stand-up (Daily Scrum)  C) Retrospective  D) Sprint Planning
**Answer: B)**

**Q6.** Which model combines iterative development with risk analysis repeated at each cycle, often drawn as a spiral?
A) Waterfall  B) Spiral Model  C) V-Model  D) Big Bang Model
**Answer: B) Spiral Model**

**Q7.** The **V-Model** is essentially an extension of which model, pairing a testing phase with each development phase?
A) Agile  B) Waterfall  C) Spiral  D) RAD
**Answer: B) Waterfall**

**Q8.** Which model emphasizes quickly building a working sample of the system to gather user feedback **before** full-scale development?
A) Prototyping Model  B) Waterfall Model  C) V-Model  D) Big Bang Model
**Answer: A) Prototyping Model**

**Q9.** RAD (Rapid Application Development) primarily focuses on:
A) Long, heavy documentation phases  B) Fast development using reusable components and rapid prototyping  C) Avoiding any user feedback  D) Strict, rigid sequential phases
**Answer: B)**

**Q10.** Which of these is **NOT** typically a phase of the general SDLC?
A) Requirements gathering  B) Design  C) Marketing the company's social media accounts  D) Testing
**Answer: C)**

**Q11.** A key disadvantage of the Waterfall model is:
A) It's too flexible for changing requirements  B) It's hard/costly to go back and change earlier phases once you've moved forward  C) It produces no documentation  D) It cannot be used for any real project
**Answer: B)**

**Q12.** A key advantage of Agile over Waterfall is:
A) Agile has zero project risk  B) Agile allows continuous feedback and adapts to changing requirements  C) Agile requires no customer involvement at all  D) Agile has no iterations
**Answer: B)**

**Q13.** In Scrum, who is primarily responsible for representing the customer's interests and prioritizing the product backlog?
A) Scrum Master  B) Product Owner  C) Development Team  D) Any random stakeholder
**Answer: B) Product Owner**

**Q14.** The **Iterative model** develops software through:
A) One single final release after every phase completes  B) Repeated cycles, each adding more functionality to a growing, working version  C) Random, unplanned changes with no structure  D) Documentation only, with no coding
**Answer: B)**

---

## Quick-recall summary

| Model | Core idea |
|---|---|
| Waterfall | Strict sequential phases, no overlap, hard to revise |
| Agile / Scrum | Short sprints, embraces changing requirements |
| Kanban | Visual board, continuous flow (not fixed sprints) |
| Spiral | Iterative + risk analysis each loop |
| V-Model | Waterfall + a paired test phase for every dev phase |
| Prototyping | Build a quick mock-up first for feedback |
| RAD | Fast, component-reuse-driven development |
| Iterative/Incremental | Repeated cycles, growing functionality each time |
