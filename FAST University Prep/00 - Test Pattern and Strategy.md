# FAST-NUCES MS Admission Test — Pattern & Strategy
**Programs:** MS Artificial Intelligence / MS Data Science — Test Date: **4 July 2026**

This file is your map of the exam. Read it first.

---

## 1. Why you don't need GAT/GRE

FAST lets you get in two ways for an MS:
1. A valid **GAT-General** (≥50th percentile) or **GRE General** score, OR
2. **FAST's own Admission Test** (this is the one you're sitting).

Graduate merit = **50% your previous degree CGPA/percentage + 50% your test score**. So the test genuinely moves your merit number — it's worth treating seriously, not just "passing."

## 2. Confirmed Official Test Pattern

Source: FAST official admissions site, "Test Pattern" page (nu.edu.pk/Admissions/TestPattern). MS in AI and MS in Data Science both fall under the **"MS (Computing)"** row (this row covers all computing-related MS degrees: CS, SE, AI, DS, Cyber Security).

| Section | Weight |
|---|---|
| English | 5% |
| Essay Writing | 10% |
| Analytical Skills & IQ | 30% |
| Specialization Area | 30% |
| Quantitative Methods | 30% |

> Note: these five numbers add to slightly over 100% as published — this is how FAST's own page lists it (likely a rounding artifact on their site), not an error on our side. Treat it as: **Analytical, Specialization, and Quantitative are roughly equal and together are ~90% of your score; English + Essay are the remaining ~10–15%.** Confirm the exact figures on your admit/roll-number slip closer to the date, since universities do revise these slightly year to year.

**Other confirmed rules (official page):**
- ❗ **Negative marking applies.** Wrong answers cost you marks — don't guess blindly.
- 🚫 **Calculators are NOT allowed.** Practice doing arithmetic by hand.
- The test is computer-based and sections may appear in a randomized/mixed order, not in neat labeled blocks — so you must recognize *what kind* of question you're looking at quickly, not rely on section headers.

## 3. Mapping your memory of last time to the official sections

You recalled: *"one essay, math question like symbols what next, sort of mcqs, square numbers, some questions seemed old."* Here's how that maps to the real, named sections — including one you likely didn't realize was its own scored category:

| What you remember | Official section | What it actually is |
|---|---|---|
| "One essay" | **Essay Writing (10%)** | A short written essay, hand-typed, on a general topic. Graded on structure, grammar, argument — not technical knowledge. See file `04`. |
| "Math question like symbols, what next" | **Analytical Skills & IQ (30%)** | Number/letter series, symbol-coded arithmetic ("if + means ×..."), pattern completion, coding-decoding, blood relations, syllogisms. This is the **single biggest section** — and the one people under-prepare for because it doesn't feel like "real math." See file `01`. |
| "Square numbers" | **Quantitative Methods (30%)** | Arithmetic, percentages, ratios, basic algebra, and number properties (squares, cubes, divisibility, primes). See file `02`. |
| "Sort of MCQs" | All sections except Essay are MCQ | Everything else is multiple choice. |
| "Some questions seemed old" | Recycled question banks | FAST's bank overlaps heavily with public NTS/GAT-style practice question pools that circulate across prep sites — hence the déjà-vu feeling. See file `06` for where that pool lives. |
| *(You didn't mention this one — but it's 30% of your score)* | **Specialization Area (30%)** | Programming logic, **Object-Oriented Programming**, and **Data Structures & Algorithms** basics. Proof this is real: FAST lists "Object Oriented Programming" and "Data Structures and Algorithms" as *prerequisite courses* for MS(AI)/MS(DS), and explicitly says the non-credit "Applied Programming" course **can be waived based on good performance in the admission test** — meaning these CS-fundamentals questions are mixed into your MCQs even if they didn't register as a separate "section" last time. See file `05`. |

## 4. Where the marks actually are — priority order

Given equal-ish weights (30/30/30) but very different prep cost, attack in this order:

1. **Analytical Skills & IQ (30%)** — highest ROI. Pure pattern-recognition skill, trains fast with practice, no prior knowledge needed. File `01`.
2. **Quantitative Methods (30%)** — mostly Matric/Inter-level arithmetic & algebra you've already learned once. Mostly about speed + accuracy without a calculator. File `02`.
3. **Specialization Area (30%)** — needs real understanding of OOP + DSA, but this is exactly the same content you need for NED's OOP/DAA sections (see the NED folder) — **double-purpose study**. File `05`.
4. **Essay Writing (10%)** — quick to raise with a memorized structure + 2–3 practiced openings. File `04`.
5. **English (5%)** — lowest weight; light daily review only. File `03`.

## 5. Test-day mechanics (because of negative marking)

- **Only answer if you can eliminate at least 2 of the 4 options.** Pure guesses on 4-option MCQs have negative expected value once a wrong answer costs more than ~0.25–0.33 of a correct mark.
- Do an **easy-question sweep first** across the whole test (since sections are mixed), flag anything that needs a second look, then return.
- For Analytical/symbol questions: **write the decoded rule down on scratch paper before computing** — most errors come from re-deriving the rule mid-calculation and slipping.
- For Quantitative: estimate first, compute second — if your gut estimate and your calculation disagree by an order of magnitude, you made an arithmetic slip (no calculator, so this catches a lot of mistakes).
- Don't spend more than ~60–90 seconds stuck on any one MCQ on first pass. Mark and move on.
